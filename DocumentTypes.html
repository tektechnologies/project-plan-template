Document Types Reference

The following are standards for various document types used for software design. A typical document name will begin with the two-letter type prefix, underscore, project name, underscore, and document name. For example, a Use Case document to describe the user printing in a project called Example would have a name of UC_Example_User_Prints.doc


Type Prefix Description
Specifications SP These documents define the scope of a project and the completion conditions. No project should begin without a specification document.
Use Case UC This describes both Use Case Diagrams and Use Case Documents. A typcial file name is UC_ProjectName_PrimaryActor_ActivityName where the activity name is the name of the action covered by the use case.
Data Dictionary DD If the project includes a list of data items to be stored, that list goes here.
SQL SQ Documents containing SQL for either DDL or DML.
Entity Relationship Diagrams ER Diagrams that describe the relationships between database entities.
State Chart SC UML style State Chart Diagrams
Class Diagrams CD UML Style Class Diagrams
Sequence Diagrams SE UML style Sequence Diagrams
Activity Diagrams AD UML style Activity Diagrams (the ones with "swim lanes")
Deployment Diagrams DP UML style Component and Deployment Diagrams. These diagrams explain how to package the software for dispersal.
Notes NT These documents carry general notes and information that does not fit into one of the other document types.
XML Documents XL Any documents used to define the XML documents needed for the project. This includes XLST, schemas, and XSL-FO as well as others.
Index IN An index document serves as a table of contents for the other documents associated with a project. The index lists all the other documents, often with hyperlinks to those documents, and includes a brief description of the item. There should be at least one index file per project, though there may be more if waranted by circumstance.
